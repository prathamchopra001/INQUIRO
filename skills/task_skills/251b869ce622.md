import numpy as np
from statsmodels.tsa.stattools import grangercausalitytests

# Sample Data (replace with your actual data)
data = pd.DataFrame({
    'learning_firms_proportion': np.random.rand(100),
    'gdp_forecast_accuracy': np.random.rand(100)
})

# Granger Causality Test
maxlag=3 # Choose an appropriate number of lags
test_result = grangercausalitytests(data[['gdp_forecast_accuracy', 'learning_firms_proportion']], maxlag=maxlag, verbose=False)

# Print p-values for each lag
for i in range(1, maxlag + 1):
    p_value = test_result[i][0]['ssr_ftest'][1]
    print(f'Lag {i}: p-value = {p_value}')

    #Interpretation: If p-value < significance level (e.g., 0.05),
    # then 'learning_firms_proportion' Granger-causes 'gdp_forecast_accuracy'

## 5. Common Pitfalls
*   **Non-Stationarity:** Ensure your time series data is stationary before applying the Granger causality test. Use differencing (`data['series'].diff().dropna()`) if necessary and test with `adfuller` from `statsmodels.tsa.stattools`.
*   **Lag Selection:** Choosing an appropriate number of lags is crucial. Too few lags may miss important causal relationships, while too many can reduce the test's power. Consider using information criteria (AIC, BIC) to help select the optimal lag order.
*   **Data Alignment:** Ensure the time series data for 'learning\_firms\_proportion' and 'gdp\_forecast\_accuracy' are properly aligned by date or time index.
*   **Interpretation:** Granger causality does not imply true causality, but rather predictive causality. It indicates whether one time series can be used to predict another.
*   **Missing Values:** Handle missing values appropriately (e.g., imputation or dropping rows) before running the test.
