## DO NOT USE

*   `VAR.fit()` without specifying the order (p) of the VAR model.
*   Ignoring the need to scale data before training VAR.
*   Using `mean_squared_error` directly on time series without aligning indices.

## Example Code
```python
import pandas as pd
import numpy as np
from statsmodels.tsa.api import VAR
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_squared_error
from statsmodels.tsa.stattools import adfuller
from sklearn.preprocessing import MinMaxScaler

# Load data (replace with your actual data loading)
data = pd.read_csv('italian_macro_data.csv', index_col='Date', parse_dates=True)

# Handle missing values (impute or drop)
data = data.fillna(data.mean())

# Check for stationarity using ADF test
def is_stationary(series):
    result = adfuller(series)
    return result[1] < 0.05 #p-value < 0.05 indicates stationarity

for col in data.columns:
    if not is_stationary(data[col]):
        data[col] = data[col].diff().dropna() #Differencing to achieve stationarity

# Scale the data
scaler = MinMaxScaler()
scaled_data = scaler.fit_transform(data)
scaled_data = pd.DataFrame(scaled_data, index=data.index, columns=data.columns)


# Split data into training and testing sets
train_data, test_data = train_test_split(scaled_data, test_size=0.2, shuffle=False)

# Train VAR model
model = VAR(train_data)
model_fitted = model.fit(1) # Specify the order (p) of the VAR model

# Make predictions
lag_order = model_fitted.k_ar
forecast_input = train_data.values[-lag_order:]
predictions = model_fitted.forecast(y=forecast_input, steps=len(test_data))
predictions_df = pd.DataFrame(predictions, index=test_data.index, columns=test_data.columns)

# Inverse transform the scaled predictions
predictions_original_scale = scaler.inverse_transform(predictions_df)
predictions_df = pd.DataFrame(predictions_original_scale, index=test_data.index, columns=test_data.columns)

test_data_original_scale = scaler.inverse_transform(test_data)
test_data = pd.DataFrame(test_data_original_scale, index=test_data.index, columns=test_data.columns)


# Evaluate the VAR model
rmse = np.sqrt(mean_squared_error(test_data, predictions_df))
print(f'VAR RMSE: {rmse}')

# Load ABM forecasts (replace with your actual ABM forecast loading)
abm_forecasts = pd.read_csv('abm_forecasts.csv', index_col='Date', parse_dates=True)

# Evaluate the ABM (assuming abm_forecasts is already on the original scale)
rmse_abm = np.sqrt(mean_squared_error(test_data, abm_forecasts))
print(f'ABM RMSE: {rmse_abm}')
```

## Common Pitfalls
*   **Non-Stationary Data:** VAR assumes stationarity.  Use ADF test and differencing to ensure data is stationary.
*   **Incorrect Lag Order:** Choosing the wrong lag order (p) for VAR can significantly impact performance. Use information criteria (AIC, BIC) to select the optimal order.
*   **Data Scaling:** For variables with vastly different scales, scaling is crucial for VAR to converge and produce meaningful results.
*   **Index Alignment:** Ensure that the indices of the test data and predictions are aligned before calculating metrics like RMSE.
*   **Forgetting Inverse Transform:** Remember to inverse transform scaled predictions back to the original scale before evaluating the model.
*   **ABM Forecast Format:** Ensure that `abm_forecasts` is a Pandas DataFrame with the same columns and index as `test_data` and contains the ABM forecasts on the *original scale*.
