## Task Description
Analyze the heterogeneity in learned pricing strategies across firms of different sizes by calculating the standard deviation of pricing markups for each size category.

## Output Format
A pandas DataFrame or dictionary containing the standard deviation of pricing markups for each firm size category.

## DO NOT USE
*   `ddof=0` in `std()` if you want the sample standard deviation (which is usually the case). The default `ddof=1` calculates the sample standard deviation, which is generally preferred for statistical analysis.
*   Loops for aggregation; use `groupby()` for efficiency and clarity.
*   Hardcoded size categories. Use a flexible binning approach.

## Example Code
```python
import pandas as pd
import numpy as np

def analyze_pricing_heterogeneity(df, size_column='employment', markup_column='pricing_markup', size_bins=[0, 10, 50, 200, 500, 1000, 10000]):
    """
    Analyzes pricing heterogeneity by firm size by calculating the standard deviation of pricing markups for each size category.

    Args:
        df (pd.DataFrame): DataFrame containing firm size and pricing markup data.
        size_column (str): Name of the column representing firm size (employment).
        markup_column (str): Name of the column representing pricing markups.
        size_bins (list): List of bin edges for categorizing firm sizes.

    Returns:
        pd.DataFrame: DataFrame containing the standard deviation of pricing markups for each firm size category.
    """

    # Create size categories using bins
    df['size_category'] = pd.cut(df[size_column], bins=size_bins, right=False)

    # Calculate the standard deviation of pricing markups for each size category
    markup_std = df.groupby('size_category')[markup_column].std()

    return markup_std.to_frame(name='markup_std')


# Example Usage (replace with your actual data)

df = pd.DataFrame(data)

result = analyze_pricing_heterogeneity(df)
```

## Common Pitfalls
*   **Incorrect Binning:** Ensure the `size_bins` cover the entire range of firm sizes in your data. Missing firms due to incorrect bins will skew the results. The `right=False` argument in `pd.cut` is important to define the bin edges correctly.
*   **Missing Data:** Handle missing values (`NaN`) in the `employment` or `pricing_markup` columns appropriately before calculating standard deviations. Consider using `df.dropna()` or imputing missing values.
*   **Small Sample Sizes:**  Be cautious when interpreting standard deviations for size categories with very few firms. The standard deviation may not be a reliable measure of heterogeneity in such cases.
*   **Data Types:** Ensure that the `employment` and `pricing_markup` columns have numeric data types. Convert them using `pd.to_numeric()` if necessary.
*   **Interpreting `NaN`:** If a group has only one element, `std()` will return `NaN`. Handle this case if necessary based on the application.
