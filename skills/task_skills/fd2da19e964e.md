## DO NOT USE
*   Avoid `ARIMA.fit().forecast()` (deprecated). Use `ARIMA.fit().get_forecast().summary_frame()` or `ARIMA.fit().forecast()` with explicit steps.
*   Do not use `sklearn.model_selection.train_test_split` directly on time series data without shuffling; time series data needs to maintain temporal order.
*   Avoid manual time series splitting without considering data leakage.

## Example Code
```python
import numpy as np
import pandas as pd
from statsmodels.tsa.arima.model import ARIMA
from sklearn.metrics import mean_squared_error
from math import sqrt

# Generate synthetic GDP data
np.random.seed(42)
gdp_data = np.random.randn(100)
gdp_series = pd.Series(gdp_data)

# Split data into training and testing sets
train_size = int(len(gdp_series) * 0.8)
train, test = gdp_series[0:train_size], gdp_series[train_size:len(gdp_series)]

# ARIMA Baseline Model
history = [x for x in train]
predictions = list()
for t in range(len(test)):
    model = ARIMA(history, order=(5,1,0))  # Example ARIMA order
    model_fit = model.fit()
    output = model_fit.forecast()
    yhat = output[0]
    predictions.append(yhat)
    obs = test[t+train_size] #Correct Indexing
    history.append(obs)

rmse = sqrt(mean_squared_error(test, predictions))
print(f'ARIMA RMSE: {rmse}')

## Common Pitfalls
*   **Incorrect Time Series Splitting:** Ensure that the training data precedes the testing data. Avoid random shuffling.
*   **Data Leakage:** Prevent information from the test set from influencing the training of the model.
*   **ARIMA Order Selection:** Choosing the appropriate (p, d, q) order for the ARIMA model is crucial. Use techniques like AIC/BIC or auto_arima.
*   **Indexing Errors:** Carefully manage indexing when iterating through time series data, especially when updating the training history. Verify that `obs = test[t+train_size]` uses the correct index.
