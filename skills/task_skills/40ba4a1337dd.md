# Skill: Identify Sector with Lowest Inflation Forecast Error Proportion

This skill focuses on identifying, for each sector, the proportion of learning firms that results in the lowest inflation forecast error.

## Task Description

Given a dataset of firms, their sector, whether they are 'learning' firms, and their inflation forecast error, the task is to:

1.  Group data by sector.
2.  For each sector, calculate the proportion of learning firms.
3.  Identify the proportion that corresponds to the minimum inflation forecast error within that sector.

## Output Format

A pandas DataFrame or Series showing the sector and the proportion of learning firms associated with the lowest inflation forecast error.

## DO NOT USE

*   Iterating directly over DataFrame rows for calculations (use vectorized operations).
*   Hardcoding sector names.
*   Using `min()` directly on a grouped object without specifying the column.

## Example Code

```python
import pandas as pd
import numpy as np

# Sample Data - Replace with your actual data loading
data = {'sector': ['A', 'A', 'B', 'B', 'A', 'B'],
        'learning': [True, False, True, True, False, False],
        'inflation_error': [0.1, 0.2, 0.05, 0.15, 0.15, 0.2]}
df = pd.DataFrame(data)

# Calculate proportion of learning firms for each sector
learning_proportions = df.groupby('sector')['learning'].mean()

# Merge proportions back into the original DataFrame
df['learning_proportion'] = df['sector'].map(learning_proportions)

# Find the minimum inflation error for each sector
min_inflation = df.groupby('sector')['inflation_error'].min()

# Filter the DataFrame to only include rows with the minimum inflation error
result = df[df['inflation_error'].isin(min_inflation)].drop_duplicates(subset=['sector'])[['sector', 'learning_proportion']]

print(result)
```

## Common Pitfalls

*   **Incorrect Aggregation:** Ensure the proportion is calculated correctly within each sector using `.mean()` on the boolean 'learning' column.
*   **Data Type Issues:** Verify the 'learning' column is boolean (True/False) or can be coerced to it.
*   **Multiple Minimums:** If multiple proportions result in the same minimum inflation error for a sector, the code returns one of them arbitrarily. Consider how to handle ties (e.g., return all).
*   **Missing Data:** Handle cases where a sector might have no learning firms (resulting in NaN).
*   **Not dropping duplicates:** If you don't drop duplicates, you might get multiple rows for the same sector that have the minimum inflation error.
