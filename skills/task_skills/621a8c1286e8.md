## Common Pitfalls
*   **Non-Stationary Data:** ARIMA and VAR assume stationarity. Use differencing or other transformations to achieve stationarity before modeling. Check using `adfuller` test.
*   **Incorrect Model Order:**  ARIMA(p, d, q) and VAR lag order (p) need to be carefully chosen using ACF, PACF plots, or information criteria (AIC, BIC).
*   **Data Leakage:** Ensure that the test set is not used during model training or parameter tuning.
*   **Scaling Issues:** If variables have vastly different scales, consider scaling them before using VAR.
*   **VAR Variables:** VAR requires multiple related time series. Ensure your data is suitable.
*   **ABM Integration:**  The example assumes you have ABM forecasts. The key is to align the forecast horizon of ABM with ARIMA/VAR for a fair comparison.
