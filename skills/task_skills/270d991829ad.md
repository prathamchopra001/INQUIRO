## DO NOT USE
*   Avoid using loops for calculating relative profit across the entire sector; use vectorized operations instead.
*   Don't ignore potential division-by-zero errors when calculating relative profit.
*   Avoid hardcoding sector IDs or peer groups; ensure flexibility.

## Example Code

```python
import numpy as np

def reward_function(price, cost, sales_volume, sector_prices, sector_costs, sector_sales_volumes, weight_absolute=0.5, weight_relative=0.5):
    """
    Calculates a reward based on absolute and relative profit.

    Args:
        price (float): The price of the product.
        cost (float): The cost of the product.
        sales_volume (float): The sales volume achieved.
        sector_prices (np.array): Array of prices of competitors in the same sector.
        sector_costs (np.array): Array of costs of competitors in the same sector.
        sector_sales_volumes (np.array): Array of sales volumes of competitors in the same sector.
        weight_absolute (float): Weight given to absolute profit (0 to 1).
        weight_relative (float): Weight given to relative profit (0 to 1).

    Returns:
        float: The calculated reward.
    """

    # Calculate absolute profit
    absolute_profit = (price - cost) * sales_volume

    # Calculate sector profit
    sector_profit = np.sum((sector_prices - sector_costs) * sector_sales_volumes)

    # Calculate relative profit
    if sector_profit > 0:
        relative_profit = absolute_profit / sector_profit
    else:
        relative_profit = 0  # Handle the case where the sector profit is zero

    # Combine absolute and relative profit
    reward = weight_absolute * absolute_profit + weight_relative * relative_profit

    return reward

# Example usage
price = 10.0
cost = 5.0
sales_volume = 100.0
sector_prices = np.array([9.0, 11.0, 10.5])
sector_costs = np.array([4.0, 6.0, 5.5])
sector_sales_volumes = np.array([110.0, 90.0, 105.0])

reward = reward_function(price, cost, sales_volume, sector_prices, sector_costs, sector_sales_volumes)
print(f"Reward: {reward}")
```

## Common Pitfalls
*   **Division by Zero:**  Ensure that the `sector_profit` is not zero before calculating `relative_profit`. Add a check to prevent `ZeroDivisionError`.
*   **Incorrect Vectorization:** Using loops to calculate sector profit. Vectorized operations with NumPy are significantly faster.
*   **Weighting:** Ensure `weight_absolute + weight_relative == 1` to properly balance the reward components.
*   **Data Types:** Ensure that price, cost, and sales volume are numeric types (float or int) to avoid type errors during calculations.
*   **Negative Sales Volumes/Prices/Costs**: Add input validation to ensure all values are valid.
