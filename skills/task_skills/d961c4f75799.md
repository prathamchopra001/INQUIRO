# Skill: Competitive Profit Maximization Reward Signal

## Task Description
Calculate a reward signal for firms in an Agent-Based Model (ABM) based on their profit relative to their sectoral peers. The reward is higher for firms with profits exceeding the sectoral average and lower for those below.

## Output Format
A numerical reward signal for each firm.

## Correct Imports
```python
import numpy as np
```

## DO NOT USE
*   Loops for calculating sectoral averages. Use NumPy's vectorized operations.
*   Hardcoding sector identifiers. The function should accept sector assignments as input.
*   Ignoring edge cases like empty sectors (handle with a default reward or zero).
*   Assuming all firms have the same production capacity or cost structure.

## Example Code
```python
import numpy as np

def competitive_profit_reward(profits, sectors):
    """
    Calculates a competitive profit maximization reward signal.

    Args:
        profits (np.ndarray): Array of firm profits.
        sectors (np.ndarray): Array of sector assignments for each firm.

    Returns:
        np.ndarray: Reward signal for each firm.
    """
    unique_sectors = np.unique(sectors)
    rewards = np.zeros_like(profits, dtype=float)

    for sector in unique_sectors:
        sector_indices = sectors == sector
        sector_profits = profits[sector_indices]

        if len(sector_profits) > 0:  # Handle empty sectors
            sector_average_profit = np.mean(sector_profits)
            rewards[sector_indices] = profits[sector_indices] - sector_average_profit
        else:
            rewards[sector_indices] = 0 # Or some default reward

    return rewards

# Example usage:
profits = np.array([100, 150, 80, 120, 90])
sectors = np.array(['A', 'A', 'B', 'B', 'B'])
rewards = competitive_profit_reward(profits, sectors)
print(rewards)
```

## Common Pitfalls
*   **Zero Division Error:** Ensure that sectors always have at least one firm before calculating averages.
*   **Incorrect Sector Assignment:** Double-check that the `sectors` array aligns correctly with the `profits` array.
*   **Data Type Mismatch:** Ensure both `profits` and `sectors` are NumPy arrays for efficient calculations. Convert if necessary.
*   **Ignoring Scale:** The magnitude of the reward might need scaling based on the ABM's specific dynamics. Consider normalizing the reward signal.
*   **Lack of Vectorization:** Failing to leverage NumPy's vectorized operations will lead to inefficient code.
