## Example Code

```python
import numpy as np
import gymnasium as gym
import random

# Simplified example - adapt to your ABM environment
class QLearningAgent:
    def __init__(self, observation_space_size, action_space_size, learning_rate=0.1, discount_factor=0.9, exploration_rate=0.1):
        self.q_table = np.zeros((observation_space_size, action_space_size))
        self.learning_rate = learning_rate
        self.discount_factor = discount_factor
        self.exploration_rate = exploration_rate

    def choose_action(self, state):
        if random.random() < self.exploration_rate:
            return random.randint(0, self.q_table.shape[1]-1) # Explore
        else:
            return np.argmax(self.q_table[state, :]) # Exploit

    def learn(self, state, action, reward, next_state):
        best_q = np.max(self.q_table[next_state, :])
        td_target = reward + self.discount_factor * best_q
        td_error = td_target - self.q_table[state, action]
        self.q_table[state, action] += self.learning_rate * td_error

# Example usage (adapt to ABM)
# Assuming your ABM provides a 'state' and 'reward' for each agent step
# and has a defined 'action_space'.

#Initialize agent
agent = QLearningAgent(observation_space_size=10, action_space_size=3) #Replace with your ABM's values

#Inside the ABM's simulation loop, for each agent:
state = 5 #Get state from ABM environment
action = agent.choose_action(state)
reward = 0.1 #Get reward from ABM environment after taking the action
next_state = 6 #Get next state from ABM environment
agent.learn(state, action, reward, next_state)

```

## Common Pitfalls
*   **Incorrect State Representation:** Ensure the state used for Q-learning accurately reflects the agent's environment in the ABM. Discretize continuous state spaces appropriately.
*   **Reward Shaping:** Define a reward function that encourages desired behavior within the ABM context.  Poorly designed rewards can lead to unintended consequences.
*   **Exploration-Exploitation Balance:** Tune the exploration rate. Too little exploration can lead to suboptimal policies; too much exploration can slow down learning. Decay exploration rate over time.
*   **ABM Integration:**  Carefully integrate the Q-learning agent's actions into the ABM's existing mechanisms.  Ensure actions have the intended effect.
*   **Non-Stationary Environment:** Be aware that the ABM environment might be non-stationary due to other agents' learning. Consider techniques like experience replay or adaptive learning rates.
