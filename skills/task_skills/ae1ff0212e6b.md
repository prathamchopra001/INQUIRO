## Example Code
```python
# Sample data (replace with your actual ABM and real data)
abm_forecasts = pd.DataFrame({
    'GDP': [17000, 17500, 18000],
    'Inflation': [2.0, 2.5, 3.0]
})
actual_values = pd.DataFrame({
    'GDP': [16800, 17600, 18200],
    'Inflation': [1.8, 2.7, 3.2]
})

# Calculate RMSE and MAE
def calculate_rmse_mae(forecast, actual):
  rmse = sqrt(mean_squared_error(actual, forecast))
  mae = mean_absolute_error(actual, forecast)
  return rmse, mae

# ARIMA benchmark (example for GDP)
history = actual_values['GDP'].iloc[:-len(abm_forecasts)] # Use pre-forecast data to train
arima_model = ARIMA(history, order=(5,1,0)) # Example order
arima_fit = arima_model.fit()
arima_forecast = arima_fit.forecast(steps=len(abm_forecasts))


# Compare ABM and ARIMA
for col in abm_forecasts.columns:
    abm_rmse, abm_mae = calculate_rmse_mae(abm_forecasts[col], actual_values[col])
    print(f"ABM {col} - RMSE: {abm_rmse:.2f}, MAE: {abm_mae:.2f}")

arima_rmse, arima_mae = calculate_rmse_mae(arima_forecast, actual_values['GDP'].iloc[-len(abm_forecasts):])
print(f"ARIMA GDP - RMSE: {arima_rmse:.2f}, MAE: {arima_mae:.2f}")
```

## Common Pitfalls
*   **Data Alignment:** Ensure ABM forecasts and actual values are aligned by date or time period.
*   **ARIMA Order Selection:**  Choosing the appropriate `order` parameter (p, d, q) for the ARIMA model is crucial. Use techniques like AIC/BIC to determine optimal values.
*   **Incorrect Data Types:** Verify that the data used for calculations are numeric (float or int). Convert if necessary.
*   **Insufficient History for ARIMA:** Ensure sufficient historical data to reliably train the ARIMA model. The `history` variable must not include values you are trying to forecast.
*   **Forgetting .iloc[] when comparing ARIMA forecast to last n actuals.**
